# 프로젝트 기획서

### **KAMCO 온비드 통합 입찰 정보 공개 및 구매 지원 플랫폼**

### **1. 프로젝트 개요**

- **프로젝트명:** **KAMCO** 온비드 통합 입찰 정보 공개 및 구매 지원 플랫폼 (가칭)
- **목표:** 한국자산관리공사(KAMCO)의 온비드 캠코공매물건 API를 활용하여, 사용자 친화적인 공매 정보 제공 및 통합 구매 지원 웹 플랫폼을 구축합니다.
- **배경:** 현재 공매 정보는 접근성이 낮거나 정보가 분산되어 있어 일반 사용자가 효율적으로 정보를 탐색하고 구매 결정을 내리기 어렵습니다. 이러한 불편함을 해소하고, 공매 시장 참여를 돕는 플랫폼을 만들고자 합니다.
- **목적:**
    - KAMCO 공매물건 정보의 접근성과 가독성을 극대화하여 사용자 편의성 증대
    - 개인 사용자에게 최적화된 맞춤형 입찰 정보 제공 및 알림 기능 구현
    - 통합 구매 프로세스 지원을 통해 사용자의 구매 결정 및 실행을 간소화
    - Java, Spring Boot, Vue.js, React, DB 경험 등 다양한 기술 스택 활용 및 포트폴리오 강화

### **2. 주요 기능 및 서비스**

- **2.1 입찰 정보 공개 웹사이트**
    - **실시간 공매물건 검색 및 조회:**
        - 지역, 물건 종류(부동산, 동산), 용도, 가격대, 진행 상태 등 다양한 조건으로 필터링 및 검색.
        - KAMCO 공매물건 정보 API를 통해 최신 공매물건 정보를 즐겨찾기 지원.
    - **물건 상세 정보 페이지:**
        - 공매 물건의 사진, 위치(지도 연동), 감정가, 최저입찰가, 입찰 기간, 진행 상태 등 상세 정보 제공.
        - 관련 서류(공고문 등) 및 법률 정보 요약 제공 (가능하다면).
        - 유사 물건 추천 기능.
    - **관심 물건 등록 및 알림:**
        - 관심 있는 물건을 등록하고, 입찰 시작/마감, 유찰/낙찰 등 주요 이벤트 발생 시 알림 (이메일, 웹 푸시).
    - **데이터 시각화:**
        - 지역별, 유형별 공매 물건 현황 및 유찰률, 낙찰가율 등 통계 데이터 시각화 제공.
- **2.2 통합 구매 웹페이지 (마이페이지 중심)**
    - **회원 가입 및 로그인:** 일반 회원 및 간편 로그인 지원. (Spring Security 활용)
    - **대시보드:**
        - 관심 물건, 참여 중인 입찰 현황, 최근 본 물건 등 개인화된 정보 요약.
    - **입찰 지원 기능:**
        - 가상 입찰 금액 계산기 (세금, 수수료 등 포함).
    - **낙찰 및 결제 관리:**
        - 낙찰 물건 관리 및 잔금 납부 안내.
        - (추후 확장) 간편 결제 시스템 연동 (예: 페이팔, 카카오페이 등).

### **3. 기술 스택**

- **백엔드:**
    - **언어:** Java
    - **프레임워크:** Spring Boot (RESTful API 설계, Spring Security 연동)
    - **데이터베이스:** Oracle (Cloud DB), MariaDB
    - **ORM/퍼시스턴스:** MyBatis (SQL 직접 제어의 유연성), JPA (빠른 개발, 복잡성 감소) – 필요시 조합 또는 선택
    - **API 연동:** RestTemplate 또는 WebClient (KAMCO API 호출)
- **프론트엔드:**
    - **프레임워크:** Vue 3 (Composition API, Pinia를 활용한 상태 관리) 또는 React (Redux/Context API 활용) – 영남님께서 선호하시는 프레임워크 선택
    - **언어:** JavaScript (타입 안정성 확보)
    - **번들러:** Vite (빠른 개발 환경)
    - **스타일링:** SCSS (유연한 스타일링)
    - **API 통신:** Axios
- **배포 및 인프라:**
    - **클라우드:** Oracle Cloud Infrastructure (OCI) - 2개 서버 활용, 로드 밸런싱
    - **웹/WAS:** Nginx (리버스 프록시, 로드 밸런싱), Tomcat (Spring Boot 애플리케이션 서빙)
    - **CI/CD:** (선택 사항, 추후 고려) Jenkins, GitHub Actions 등

### **4. 기대 효과**

- **개발자 역량 강화:** Java, Spring Boot, Vue.js/React, JavaScript, 데이터베이스 연동, API 설계 및 통합 등 풀스택 개발 역량을 종합적으로 발휘하고 심화할 수 있는 좋은 기회.
- **실제 서비스 경험:** 공공 API를 활용하여 실제 가치를 창출하는 서비스를 기획하고 개발, 운영하는 경험.
- **포트폴리오 구축:** 기술적 깊이와 문제 해결 능력을 효과적으로 보여줄 수 있는 강력한 포트폴리오.
- **시장 이해도 증진:** 공매 시장에 대한 이해를 높이고, 데이터 기반의 서비스 기획 능력을 향상시킬 수 있습니다.

### **5. 추진 전략 및 로드맵**

- **1단계 (1~2주): 기획 및 API 분석**
    - 상세 기능 정의 및 사용자 스토리 작성.
    - KAMCO API 문서 심층 분석 및 데이터 구조 이해.
    - 기술 스택 최종 확정 및 개발 환경 세팅.
- **2단계 (2~4주): 핵심 기능 개발**
    - **백엔드:** Spring Boot 기반 RESTful API 개발 (데이터베이스 연동, KAMCO API 호출 및 데이터 가공).
    - **프론트엔드:** Vue 3/React 기반 UI/UX 설계 및 핵심 페이지(메인, 검색 결과, 상세 페이지) 구현.
    - 회원가입/로그인 및 인증(Spring Security, JWT 등) 구현.
- **3단계 (4~6주): 기능 확장 및 고도화**
    - 관심 물건 등록 및 알림 기능 구현.
    - 통합 구매 웹페이지 기능 구현 (대시보드, 입찰 지원 등).
    - 배포 환경 구축 및 클라우드(OCI) 설정.
- **4단계 (6주 이후): 테스트 및 배포**
    - 통합 테스트, 버그 수정 및 성능 최적화.
    - 최종 배포 및 지속적인 기능 개선.

### **6. 차별점**

- **사용자 중심 UI/UX:** 단순 정보 나열이 아닌, 사용자가 직관적으로 정보를 얻고 구매 결정을 내릴 수 있도록 최적화된 인터페이스 제공.
- **개인화된 정보:** 관심 물건, 맞춤 알림, 대시보드를 통해 개인에게 의미 있는 정보만 선별하여 제공.
- **통합 구매 지원:** 정보 탐색부터 입찰 및 결제까지의 과정을 한 플랫폼에서 지원하여 편의성 극대화 (최종 목표).
